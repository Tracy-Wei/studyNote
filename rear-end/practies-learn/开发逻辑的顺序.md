# .NET Web API 新增接口开发流程 (仅作参考)

> 适用于 ASP.NET Core + EF Core（或 Repository）项目。

---

# 1. 明确需求

## 目标

在开发之前，需要明确：

- 接口用途
- 请求方式（GET、POST、PUT、DELETE）
- 请求路径（Route）
- 请求参数
- 返回数据
- 是否需要登录验证
- 是否需要权限控制
- 是否需要记录日志

例如：

```
POST /api/attendance/check-in
```

请求：

```json
{
  "latitude": 22.123,
  "longitude": 113.456,
  "remark": "到公司"
}
```

返回：

```json
{
  "success": true
}
```

---

# 2. 分析数据库

确定：

- 是否需要新增数据表
- 是否需要新增字段
- 是否需要新增索引
- 是否需要建立外键关系

如果只是读取已有数据，可以跳过此步骤。

---

# 3. 修改 Entity（实体）

如果数据库结构发生变化：

修改 Entity。

例如：

```csharp
public class Attendance
{
    public Guid Id { get; set; }

    public Guid UserId { get; set; }

    public DateTime CheckInTime { get; set; }

    public string? Remark { get; set; }
}
```

---

# 4. 创建 Migration（如果数据库结构变化）

生成 Migration

```bash
dotnet ef migrations add AddAttendance
```

更新数据库

```bash
dotnet ef database update
```

确认数据库已成功生成对应表或字段。

---

# 5. 创建 DTO（Request / Response）

DTO 用于接口收发数据。

例如：

```
Dtos
├── CheckInRequest.cs
└── CheckInResponse.cs
```

Request：

```csharp
public class CheckInRequest
{
    public double Latitude { get; set; }

    public double Longitude { get; set; }

    public string? Remark { get; set; }
}
```

Response：

```csharp
public class CheckInResponse
{
    public Guid Id { get; set; }

    public DateTime CheckInTime { get; set; }
}
```

---

# 6. 定义 Service Interface

例如：

```
Services
└── IAttendanceService.cs
```

```csharp
public interface IAttendanceService
{
    Task<CheckInResponse> CheckInAsync(CheckInRequest request);
}
```

# 7. 编写 Service

实现业务逻辑。

例如：

```
AttendanceService.cs
```

业务一般包括：

- 参数校验
- 查询数据库
- 判断业务规则
- 创建实体
- 保存数据库
- 返回结果

例如：

```csharp
public async Task<CheckInResponse> CheckInAsync(CheckInRequest request)
{
    // 判断今天是否已签到

    // 判断是否迟到

    // 创建 Attendance Entity

    // 保存数据库

    // 返回 Response
}
```

# 8. Repository（如果项目使用 Repository）

如果项目采用 Repository 模式：

新增：

```
IAttendanceRepository
AttendanceRepository
```

例如：

```csharp
await _attendanceRepository.InsertAsync(entity);
```

或者

```csharp
await _attendanceRepository.AddAsync(entity);
```

如果项目直接使用 DbContext，可跳过 Repository。

---

# 9. 保存数据库

EF Core：

```csharp
_db.Attendances.Add(entity);

await _db.SaveChangesAsync();
```

Repository：

```csharp
await repository.InsertAsync(entity);

await unitOfWork.SaveChangesAsync();
```

---

# 10. 编写 Controller

Controller 负责：

- 接收请求
- 调用 Service
- 返回结果

例如：

```csharp
[ApiController]
[Route("api/attendance")]
public class AttendanceController : ControllerBase
{
    private readonly IAttendanceService _attendanceService;

    [HttpPost("check-in")]
    public async Task<IActionResult> CheckIn(CheckInRequest request)
    {
        var result = await _attendanceService.CheckInAsync(request);

        return Ok(result);
    }
}
```

---

# 11. 配置依赖注入（如果需要）

如果项目没有自动注册。

Program.cs

```csharp
builder.Services.AddScoped<IAttendanceService, AttendanceService>();

builder.Services.AddScoped<IAttendanceRepository, AttendanceRepository>();
```

# 12. 权限控制

根据需求添加：

例如：

```csharp
[Authorize]
```

或者：

```csharp
[Authorize(Roles = "Admin")]
```

---

# 13. 参数校验

例如：

```csharp
public class CheckInRequest
{
    [Required]
    public double Latitude { get; set; }

    [Required]
    public double Longitude { get; set; }
}
```

或者使用 FluentValidation。

---

# 14. 异常处理

业务异常：

```csharp
throw new BusinessException("今天已经签到");
```

或者返回：

```csharp
return BadRequest("今天已经签到");
```

# 核心职责划分

| 层级       | 职责                               |
| ---------- | ---------------------------------- |
| Controller | 接收请求、返回响应，不处理业务逻辑 |
| DTO        | 定义接口输入输出模型               |
| Service    | 编写业务逻辑、业务校验             |
| Repository | 数据访问（可选）                   |
| DbContext  | 操作数据库                         |
| Entity     | 数据库实体映射                     |
| Migration  | 管理数据库结构变更                 |
| Database   | 持久化存储                         |
